# Bilingual Rules / バイリンガルルール

All community health files must be bilingual (English / Japanese).  
すべてのコミュニティヘルスファイルは英日バイリンガルであること。

All rules are written as Yes/No checks.  
すべてのルールはYes/Noで判定できる条件として記述。

---

## Structure / 構造

- [ ] English content appears **before** Japanese in every section?  
      すべてのセクションで英語が日本語より**前**に来るか？
- [ ] Each English paragraph/sentence is immediately followed by its Japanese equivalent?  
      各英語段落・文の直後に日本語訳があるか？
- [ ] Section headings are bilingual, slash-separated? (e.g., `## Overview / 概要`)  
      セクション見出しはスラッシュ区切りのバイリンガル形式か？（例：`## Overview / 概要`）

---

## Issue & PR Templates / Issue・PRテンプレート

- [ ] Template `name:` field is bilingual? (e.g., `Bug Report / バグ報告`)  
      テンプレートの `name:` フィールドがバイリンガルか？
- [ ] Template `about:` field is bilingual?  
      `about:` フィールドがバイリンガルか？
- [ ] All section headings inside the template are bilingual?  
      テンプレート内のすべてのセクション見出しがバイリンガルか？
- [ ] Inline HTML comments use `<!-- EN / JA -->` format?  
      インラインHTMLコメントは `<!-- EN / JA -->` 形式を使っているか？

**Good comment format / 良いコメント例:**
```markdown
<!-- Describe the bug clearly. / バグを明確に説明してください。 -->
```

---

## Tone & Register / 語調・敬体

- [ ] English uses plain, professional language (no slang or overly casual tone)?  
      英語はシンプルで専門的な言葉遣いか（スラングや過度なカジュアル表現はないか）？
- [ ] Japanese uses `ください` / `してください` form (丁寧語), not plain form?  
      日本語は `ください` / `してください` の丁寧語を使っているか（普通体でないか）？
